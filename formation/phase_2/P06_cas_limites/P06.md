## Le QUOI — Définition et concept

Les **cas limites** (ou *edge cases*) sont les situations non standard qui doivent être gérées sans crasher le programme. Un jeu robuste définit explicitement ce qui se passe dans chaque cas.

| Cas limite | Déclencheur | Effet sur l'état |
|-----------|------------|------------------|
| Collision drone-tempête | `drone.col == tempete.col AND drone.lig == tempete.lig` | `drone['bloque'] = 2` |
| Batterie épuisée en vol | `drone['batterie'] <= 0` après déplacement | `drone['hors_service'] = True` |
| Zone X | `(col_cible, lig_cible) in etat['zones_x']` | coût batterie +2 supplémentaire |
| Phase météo | automatique en fin de tour | chaque tempête bouge avec 50% de chance |
| Propagation zones X | tous les N tours | nouvelles cases X autour des X existantes |
| Fin de partie | conditions vérifiées | `etat['partie_finie'] = True` |

## Le POURQUOI — Enjeu pédagogique

Les cas limites sont la frontière entre un prototype qui marche « quand tout va bien » et un programme utilisable en conditions réelles. Ignorer un cas limite ne fait pas disparaître la situation — elle arrive quand même, et le programme se comporte de façon imprévisible.

La clé : **identifier tous les cas avant de coder**, pas après. Un tableau exhaustif comme celui ci-dessus est le premier outil du développeur, pas le débogueur.

## Les RISQUES

### Si on ne le fait pas
Le jeu plante ou produit des comportements absurdes en partie réelle : un drone avec batterie négative qui continue de jouer, une tempête qui ne bouge jamais, des zones X qui ne se propagent pas.

### Si on le fait mal
**Bug classique 1 — collision :** tester collision **après** le déplacement mais **avant** de mettre à jour `bloque`. Si on oublie de vérifier `drone['bloque'] > 0` dans `valider_mouvement()`, un drone bloqué peut quand même jouer.

```python
# ERREUR : on bloque mais on oublie de vérifier dans valider_mouvement()
drone["bloque"] = 2
# valider_mouvement() ne teste pas drone['bloque'] → drone joue quand même

# CORRECT : valider_mouvement() doit commencer par :
if drone["bloque"] > 0:
    return False, f"{id_drone} est bloqué ({drone['bloque']} tours restants)"
```

**Bug classique 2 — hors_service :** ne pas vérifier `drone['hors_service']` en début de `valider_mouvement()` → un drone hors service peut encore se déplacer.

### Vérifier collision drone-tempête
```python
def verifier_collision(etat, id_drone):
    """
    Après déplacement, vérifie si le drone est sur une tempête.
    Si oui, bloque le drone 2 tours.
    Retourne un str de log.
    """
    drone = etat["drones"][id_drone]
    for tid, tempete in etat["tempetes"].items():
        if tempete["col"] == drone["col"] and tempete["lig"] == drone["lig"]:
            drone["bloque"] = 2
            return f"{id_drone} BLOQUÉ par {tid} — 2 tours"
    return ""
```

### Phase météo automatique
```python
import random

def deplacer_tempetes(etat):
    """
    Phase météo : chaque tempête a 50% de chance de se déplacer
    d'une case dans sa direction (dx, dy). Rebondit sur les bords.
    """
    TAILLE = 12
    for tid, t in etat["tempetes"].items():
        if random.random() < 0.5:
            etat["grille"][t["lig"]][t["col"]] = "."   # effacer
            t["col"] = max(0, min(TAILLE - 1, t["col"] + t["dx"]))
            t["lig"] = max(0, min(TAILLE - 1, t["lig"] + t["dy"]))
            etat["grille"][t["lig"]][t["col"]] = tid   # marquer
```

### Propagation des zones X
```python
def propager_zones_x(etat, probabilite=0.3):
    """
    Propage les zones de danger : chaque zone X a une chance
    de contaminer ses voisins directs (haut, bas, gauche, droite).
    """
    TAILLE = 12
    nouvelles = set()
    for (col, lig) in list(etat["zones_x"]):
        for dcol, dlig in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
            nc, nl = col + dcol, lig + dlig
            if 0 <= nc < TAILLE and 0 <= nl < TAILLE:
                if random.random() < probabilite:
                    nouvelles.add((nc, nl))
    etat["zones_x"] |= nouvelles
    for (col, lig) in nouvelles:
        if etat["grille"][lig][col] == ".":
            etat["grille"][lig][col] = "X"
```

### Vérification de fin de partie
```python
def verifier_fin_partie(etat, TOURS_MAX=50):
    """
    Vérifie les conditions de fin de partie.
    Modifie etat['partie_finie'] et etat['victoire'].
    """
    survivants = etat["survivants"]
    drones = etat["drones"]

    # Victoire : tous les survivants sont sauvés
    if all(s["etat"] == "sauve" for s in survivants.values()):
        etat["partie_finie"] = True
        etat["victoire"] = True
        return "Victoire ! Tous les survivants sont sauvés."

    # Défaite : tous les drones hors service
    if all(d["hors_service"] for d in drones.values()):
        etat["partie_finie"] = True
        etat["victoire"] = False
        return "Défaite — tous les drones sont hors service."

    # Défaite : nombre maximum de tours atteint
    if etat["tour"] >= TOURS_MAX:
        etat["partie_finie"] = True
        etat["victoire"] = False
        return f"Défaite — {TOURS_MAX} tours dépassés."

    return ""
```
