## Le QUOI — Définition et concept

Une **fonction d'exécution** modifie l'état du jeu et retourne un message de log décrivant ce qui s'est passé. Elle ne vérifie pas si le mouvement est légal — c'est le rôle de `valider_mouvement()` (P04). Elle fait confiance à l'entrée et applique les effets.

Le **chemin nominal** couvre les cas les plus fréquents :
1. Le drone se déplace d'une case vers la cible
2. La batterie diminue de 1 (ou +1 si transport de survivant)
3. Si le drone arrive sur un survivant → il l'embarque
4. Si le drone arrive à l'hôpital avec un survivant → livraison, score +1
5. Si le drone reste stationnaire ou arrive à l'hôpital → recharge +3 batterie

Les cas limites (collision, hors_service, zones X) sont traités dans P06.

## Le POURQUOI — Enjeu pédagogique

Séparer validation et exécution est un principe fondamental de la programmation robuste. Si `executer_mouvement()` devait aussi valider, elle deviendrait une fonction qui fait deux choses à la fois — difficile à tester, difficile à déboguer. Cette séparation garantit que chaque fonction a **une seule responsabilité**.

De plus, `executer_mouvement()` est le cœur mécanique du jeu : tout ce que fait J1 passe par elle. La comprendre parfaitement, c'est comprendre comment le jeu « vit ».

## Les RISQUES

### Si on ne le fait pas
Sans `executer_mouvement()`, le jeu ne peut pas progresser : aucun drone ne bouge, aucun survivant n'est sauvé, la boucle de jeu est inerte.

### Si on le fait mal
**Bug classique :** oublier de mettre à jour `etat["grille"]` après avoir modifié `etat["drones"][id_drone]["col"]` et `"lig"`. Résultat : les drones semblent se déplacer dans les dictionnaires mais la grille affichée est figée — le drone apparaît toujours à sa position d'origine.

```python
# ERREUR fréquente — mise à jour partielle
drone["col"] = col_cible
drone["lig"] = lig_cible
# OUBLI : mettre à jour etat["grille"] !

# CORRECT
col_avant, lig_avant = drone["col"], drone["lig"]
etat["grille"][lig_avant][col_avant] = "."
drone["col"], drone["lig"] = col_cible, lig_cible
etat["grille"][lig_cible][col_cible] = id_drone
```

## Lien avec jeu/

Extrait commenté de `jeu/logique.py` :

```python
def executer_mouvement(etat, id_drone, col_cible, lig_cible):
    drone = etat["drones"][id_drone]          # 1. récupérer le dict du drone
    col_avant, lig_avant = drone["col"], drone["lig"]

    # 2. Effacer l'ancienne position sur la grille
    etat["grille"][lig_avant][col_avant] = "."

    # 3. Déplacer le drone
    drone["col"], drone["lig"] = col_cible, lig_cible

    # 4. Calculer le coût batterie (cas nominal : 1 déplacement)
    cout = 1
    if drone["survivant"] is not None:
        cout += 1                              # +1 si transport d'un survivant
    drone["batterie"] -= cout

    # 5. Gérer prise de survivant
    for sid, surv in etat["survivants"].items():
        if surv["col"] == col_cible and surv["lig"] == lig_cible \
                and surv["etat"] == "en_attente":
            drone["survivant"] = sid
            surv["etat"] = "embarque"
            break

    # 6. Gérer livraison à l'hôpital
    hop_col, hop_lig = etat["hopital"]
    if col_cible == hop_col and lig_cible == hop_lig \
            and drone["survivant"] is not None:
        sid = drone["survivant"]
        etat["survivants"][sid]["etat"] = "sauve"
        drone["survivant"] = None
        etat["score"] += 1
        drone["batterie"] = min(drone["batterie"] + 3, drone["batterie_max"])

    # 7. Recharge si stationnaire
    if col_cible == col_avant and lig_cible == lig_avant:
        drone["batterie"] = min(drone["batterie"] + 3, drone["batterie_max"])

    # 8. Marquer la nouvelle position sur la grille
    etat["grille"][lig_cible][col_cible] = id_drone

    return f"{id_drone} déplacé en ({col_cible},{lig_cible}), batterie={drone['batterie']}"
```
