---

## Le QUOI — Définition et concept

Une **configuration externe** regroupe toutes les valeurs qui peuvent changer
sans qu'on ait besoin de modifier le code : taille de grille, nombre de drones,
batterie maximale, durée d'une partie...

En Python, on utilise `json.load()` pour lire un fichier `.json` :

```python
import json

with open("config.json", encoding="utf-8") as f:
    donnees = json.load(f)  # retourne un dict Python

GRILLE_TAILLE = donnees["GRILLE_TAILLE"]  # 12
```

### Principe DRY (Don't Repeat Yourself)

Sans configuration externe, la valeur `12` apparaît à une dizaine d'endroits
dans le code. Modifier la taille de la grille à 10 nécessite de retrouver
toutes ces occurrences. Avec `config.json`, une seule modification suffit.

---

## Le POURQUOI — Enjeu pédagogique

La configuration externe est la première leçon d'**architecture** : séparer
les données du comportement. Un programme bien structuré ne contient aucune
*valeur magique* (`12`, `20`, `3`...) directement dans le code.

Cette habitude se retrouve dans tous les projets professionnels : fichiers
`.env`, `settings.py`, `appsettings.json`... La forme change, le principe reste.

---

## Les RISQUES

### Si on ne le fait pas
Tu changes la taille de grille de 12 à 10 pour un test. Tu passes 20 minutes
à chercher toutes les occurrences de `12` dans le code, dont certaines signifient
autre chose (indices, nombres de tours...). Tu en oublies une et le jeu plante.

### Si on le fait mal
Tu lis `config.json` dans **chaque fichier** qui en a besoin :
`logique.py` lit le fichier, `affichage.py` lit le fichier, `console.py` lit
le fichier... Trois lectures inutiles. La bonne approche : lire une seule fois
dans `config.py` et importer les constantes depuis les autres modules.

## Lien avec jeu/

```python
# jeu/config.py — version complète
import json
import os

# Chemin absolu vers config.json (robuste peu importe le répertoire courant)
_DOSSIER = os.path.dirname(os.path.abspath(__file__))
_CHEMIN_CONFIG = os.path.join(_DOSSIER, "config.json")

def charger_config():
    """Lit config.json et retourne un dict de configuration."""
    with open(_CHEMIN_CONFIG, encoding="utf-8") as f:
        return json.load(f)

# Constantes exposées — importées par les autres modules
CONFIG = charger_config()
GRILLE_TAILLE   = CONFIG["GRILLE_TAILLE"]    # 12
BATTERIE_MAX    = CONFIG["BATTERIE_MAX"]      # 20
NB_DRONES       = CONFIG["NB_DRONES"]         # 2
MAX_TOURS       = CONFIG["MAX_TOURS"]         # 30
MAX_DEPL_J1     = CONFIG["MAX_DEPL_J1"]       # 3
MAX_DEPL_J2     = CONFIG["MAX_DEPL_J2"]       # 2
```

Les autres modules importent simplement :
```python
from config import GRILLE_TAILLE, BATTERIE_MAX
```
