## Objectif

À la fin de ce module, tu sais **lire un fichier JSON avec Python**, **exposer ses valeurs comme des constantes nommées** dans un module, et **importer ces constantes dans les autres fichiers** du projet sans jamais écrire de valeur magique.

---

## Le QUOI — Définition et concept

Une **valeur magique** (ou *magic number*) est un nombre ou une chaîne écrit directement dans le code sans nom explicite :

```python
# ❌ Valeur magique : que veut dire 12 ? 20 ? 40 ?
if etat["tour"] > 40:
    ...
batterie = 20
grille = [['.' for _ in range(12)] for _ in range(12)]
```

Quand les règles du jeu changent — la grille passe de 12×12 à 8×8, ou le nombre de tours maximum passe de 40 à 30 — il faut fouiller **tout le code** pour trouver et remplacer chaque occurrence. Une seule oubliée crée un bug.

La solution : **tout centraliser dans un fichier de configuration externe** (`config.json`), puis **lire ce fichier une seule fois** dans un module Python (`config.py`) qui expose des constantes nommées :

```python
# ✅ Constante nommée : le sens est clair, la valeur est unique
if etat["tour"] > NB_TOURS_MAX:
    ...
batterie = BATTERIE_MAX
grille = [['.' for _ in range(GRILLE_TAILLE)] for _ in range(GRILLE_TAILLE)]
```

Changer `NB_TOURS_MAX` dans `config.json` suffit pour mettre à jour tout le jeu.

---

## Le POURQUOI — Enjeu pédagogique

`config.py` est le **premier module à écrire** car tous les autres en dépendent. Sans lui, on ne peut pas coder `logique.py`, `affichage.py` ou `console.py`. C'est aussi le module le plus court et le plus simple : il ne contient aucune logique, seulement de la lecture et de l'exposition de constantes.

C'est ici que tu apprends deux notions fondamentales :
1. **Le principe DRY** (*Don't Repeat Yourself*) : une information est stockée à un seul endroit.
2. **La séparation code / configuration** : le code contient la *logique*, le JSON contient les *paramètres*.

---

## Les RISQUES

### Si on ne le fait pas
Toutes les constantes sont éparpillées dans le code. Modifier les règles du jeu (taille de grille, batterie max, nombre de tours) nécessite de rechercher et remplacer des dizaines d'occurrences. Une seule oublée crée un comportement incohérent difficile à détecter.

### Si on le fait mal

**Chemin relatif fragile** : si tu écris `open("config.json")` sans préciser le chemin absolu, Python cherche le fichier dans le *répertoire courant*, pas dans le dossier de `config.py`. Le code fonctionnera si tu lances depuis `jeu/`, mais échouera depuis n'importe quel autre répertoire.

```python
# ❌ Fragile : dépend du répertoire de lancement
with open("config.json") as f:
    _cfg = json.load(f)

# ✅ Robuste : chemin calculé à partir de l'emplacement de config.py lui-même
import os
_DOSSIER = os.path.dirname(os.path.abspath(__file__))
_CHEMIN_CONFIG = os.path.join(_DOSSIER, "config.json")
with open(_CHEMIN_CONFIG, encoding="utf-8") as f:
    _cfg = json.load(f)
```

## Lien avec `jeu/`

Voici le fichier `jeu/config.json` réel du jeu :

```json
{
  "grille": { "lignes": 12, "colonnes": 12 },
  "drones": { "nb_max_j1": 6, "batterie_max": 20, "batterie_depart": 10 },
  "tempetes": { "nb_max_j2": 4, "prob_meteo": 0.5 },
  "regles": {
    "cout_transport_survivant": 2,
    "cout_entree_zone_x": 2,
    "recharge_hopital_par_tour": 3
  },
  "lettres": ["A","B","C","D","E","F","G","H","I","J","K","L"],
  "directions": {
    "N": [-1,0], "S": [1,0], "E": [0,1], "O": [0,-1],
    "NE": [-1,1], "NO": [-1,-1], "SE": [1,1], "SO": [1,-1]
  }
}
```

Et l'extrait correspondant de `jeu/config.py` (lignes 1–43) :

```python
import json
import os

_DOSSIER = os.path.dirname(os.path.abspath(__file__))
_CHEMIN_CONFIG = os.path.join(_DOSSIER, "config.json")

with open(_CHEMIN_CONFIG, encoding="utf-8") as _f:
    _cfg = json.load(_f)

# ── Grille ───────────────────────────────────────────────────────────────────
GRILLE_TAILLE = _cfg["grille"]["lignes"]     # grille carrée 12×12

# ── Drones ───────────────────────────────────────────────────────────────────
NB_DRONES     = _cfg["drones"]["nb_max_j1"]
BATTERIE_MAX  = _cfg["drones"]["batterie_max"]
BATTERIE_INIT = _cfg["drones"]["batterie_depart"]

# ── Tempêtes ──────────────────────────────────────────────────────────────────
NB_TEMPETES   = _cfg["tempetes"]["nb_max_j2"]
PROB_METEO    = _cfg["tempetes"]["prob_meteo"]

# ── Règles de jeu ─────────────────────────────────────────────────────────────
COUT_TRANSPORT   = _cfg["regles"]["cout_transport_survivant"]
COUT_ZONE_X      = _cfg["regles"]["cout_entree_zone_x"]
RECHARGE_HOPITAL = _cfg["regles"]["recharge_hopital_par_tour"]
```

Les noms commençant par `_` (`_cfg`, `_DOSSIER`) sont des **variables privées** : utiles uniquement pendant le chargement, elles ne doivent pas être importées par les autres modules.
