## Le QUOI — Définition et concept

Une **fonction de validation** répond à une question oui/non : *« Ce mouvement est-il autorisé ? »*. Elle reçoit l'état actuel et les paramètres du mouvement, et retourne un tuple `(valide, message)` :

```python
def valider_mouvement(etat, drone_id, col_cible, lig_cible):
    # ...
    return True, ""              # mouvement valide
    return False, "Hors grille"  # mouvement invalide + raison
```

Les cas à vérifier dans l'ordre :
1. Le drone existe et n'est pas hors service
2. La distance Chebyshev ≤ 1 (max 1 case, diagonales incluses)
3. La position cible est dans la grille (0 ≤ col, lig < GRILLE_TAILLE)
4. La position cible n'est pas un bâtiment (`'B'`)
5. La batterie est suffisante (≥ 1 pour se déplacer)

**Distance Chebyshev** entre deux cases : `max(|col2-col1|, |lig2-lig1|)`. C'est la distance de déplacement d'un roi aux échecs — les diagonales comptent comme 1.

```python
# Distance Chebyshev
distance = max(abs(col_cible - drone["col"]), abs(lig_cible - drone["lig"]))
if distance > 1:
    return False, "Distance trop grande"
```

## Le POURQUOI — Enjeu pédagogique

Séparer **validation** et **exécution** est l'un des principes les plus importants de conception logicielle. Si `executer_mouvement()` valide elle-même ses paramètres, il devient impossible de vérifier un mouvement sans le réaliser. L'interface (`console.py`) doit pouvoir appeler `valider_mouvement()` avant de demander à l'utilisateur de confirmer, sans modifier l'état.

Une fonction de validation **pure** (sans effet de bord) est aussi bien plus facile à tester : on lui passe un état et des paramètres, on vérifie le tuple retourné, sans jamais risquer de corrompre l'état.

## Les RISQUES

### Si on ne le fait pas
La validation est dispersée dans `executer_mouvement()`, `phase_j1()`, et ailleurs. Pour modifier une règle (ex : augmenter la distance max à 2), il faut retrouver tous les endroits où la règle est vérifiée.

### Si on le fait mal

**Modification de l'état dans la validation** : le piège classique.

```python
# ❌ Modification de l'état : INTERDIT dans une fonction de validation
def valider_mouvement(etat, drone_id, col_cible, lig_cible):
    drone = etat["drones"][drone_id]
    drone["batterie"] -= 1  # ❌ modification de l'état !
    if drone["batterie"] < 0:
        return False, "Batterie insuffisante"
    # ...

# ✅ Lire seulement, ne jamais écrire
def valider_mouvement(etat, drone_id, col_cible, lig_cible):
    drone = etat["drones"][drone_id]
    if drone["batterie"] < 1:  # lecture seule
        return False, "Batterie insuffisante"
    # ...
```

**Ordre des vérifications** : vérifier `drone_id in etat["drones"]` avant d'accéder à `etat["drones"][drone_id]`, sinon `KeyError`.

```python
# ❌ KeyError si drone_id n'existe pas
drone = etat["drones"][drone_id]
if drone["hors_service"]:
    return False, "Hors service"

# ✅ Vérification d'existence en premier
if drone_id not in etat["drones"]:
    return False, f"Drone {drone_id} inexistant"
drone = etat["drones"][drone_id]
```

## Lien avec `jeu/`

Extrait de `jeu/logique.py` (version réelle) :

```python
# jeu/logique.py — valider_mouvement()

def valider_mouvement(etat, drone_id, col_cible, lig_cible):
    """
    Vérifie si le mouvement est légal.
    Retourne (True, "") si valide, (False, message) sinon.
    Ne modifie jamais etat.
    """
    # 1. Drone existe ?
    if drone_id not in etat["drones"]:
        return False, f"Drone {drone_id} inexistant"
    drone = etat["drones"][drone_id]

    # 2. Drone opérationnel ?
    if drone["hors_service"]:
        return False, f"{drone_id} est hors service"
    if drone["bloque"] > 0:
        return False, f"{drone_id} est bloqué ({drone['bloque']} tour(s))"

    # 3. Distance Chebyshev ≤ 1
    distance = max(abs(col_cible - drone["col"]), abs(lig_cible - drone["lig"]))
    if distance > 1:
        return False, f"Distance {distance} > 1 case autorisée"

    # 4. Cible dans la grille
    if not (0 <= col_cible < GRILLE_TAILLE and 0 <= lig_cible < GRILLE_TAILLE):
        return False, "Position hors grille"

    # 5. Cible non bloquée par un bâtiment
    if etat["grille"][lig_cible][col_cible] == 'B':
        return False, "Bâtiment infranchissable"

    # 6. Batterie suffisante
    if drone["batterie"] < 1:
        return False, "Batterie insuffisante"

    return True, ""
```
