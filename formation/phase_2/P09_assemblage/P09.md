## Objectif
L'étudiant est capable d'écrire `main.py` en appelant dans l'ordre les fonctions issues des modules précédents, de comprendre le rôle de la garde `if __name__ == '__main__'`, et de lancer une partie complète depuis le terminal.

## Le QUOI — Définition et concept
`main.py` est le **chef d'orchestre** du programme. Il ne contient aucune logique métier : il se contente d'appeler les fonctions déjà écrites dans les autres modules dans le bon ordre.

Une partie se déroule ainsi :
1. Démarrer le logger → `demarrer_log()`
2. Initialiser la partie → `etat = initialiser_partie()`
3. Afficher la grille initiale → `render_grille(etat)`
4. Boucler les tours → `boucle_de_jeu(etat)` (elle-même appelle `phase_j1`, `phase_j2`, `fin_de_tour`)
5. Afficher le résultat final

```python
# jeu/main.py
from logique import initialiser_partie
from console import boucle_de_jeu
from affichage import render_grille
from logger import demarrer_log

def main():
    demarrer_log()
    etat = initialiser_partie()
    render_grille(etat)
    boucle_de_jeu(etat)
    if etat["victoire"]:
        print("Félicitations ! Tous les survivants ont été sauvés.")
    else:
        print("Partie terminée. Les drones n'ont pas pu sauver tout le monde.")

if __name__ == '__main__':
    main()
```

## Le POURQUOI — Enjeu pédagogique
Sans point d'entrée clair, un programme multi-fichiers n'a pas de début ni de fin définis. `main.py` fixe ce point d'entrée. La garde `if __name__ == '__main__'` est essentielle : elle permet d'importer `main.py` dans un test sans déclencher automatiquement l'exécution du jeu.

## Les RISQUES
### Si on ne le fait pas
Sans `main.py` structuré, le code s'exécute au niveau module dès qu'on l'importe : les tests unitaires déclenchent une partie, l'ordre d'initialisation est aléatoire, et les effets de bord sont incontrôlables.

### Si on le fait mal
**Bug typique** : mettre de la logique directement dans `main.py` plutôt que dans `console.py` ou `logique.py`. Résultat : on ne peut plus tester la logique de jeu sans lancer une vraie partie interactive.

```python
# ❌ NE PAS FAIRE
if __name__ == '__main__':
    etat = initialiser_partie()
    # ... 80 lignes de logique directement ici ...

# ✅ FAIRE
if __name__ == '__main__':
    main()  # toute la logique est dans les autres modules
```

## Lien avec jeu/
```python
# jeu/main.py — lignes 1-20 (version réelle)
from logique import initialiser_partie
from console import boucle_de_jeu
from affichage import render_grille
from logger import demarrer_log

def main():
    """Point d'entrée principal du jeu Drone Rescue."""
    demarrer_log()             # initialise le fichier de log
    etat = initialiser_partie()  # construit l'état initial
    render_grille(etat)         # affiche la grille de départ
    boucle_de_jeu(etat)         # lance la boucle principale
    # affichage du résultat final
    if etat["victoire"]:
        print("\n🏆 Victoire ! Tous les survivants sont sauvés.")
    else:
        print("\n💀 Défaite. La mission a échoué.")

if __name__ == '__main__':
    main()
```
