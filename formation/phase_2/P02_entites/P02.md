## Le QUOI — Définition et concept

Une **fonction factory** est une fonction qui **retourne un dictionnaire
initialisé** représentant une entité. Elle centralise la création et garantit
que toutes les clés obligatoires sont présentes.

```python
# Sans factory — risky : on peut oublier une clé
drone = {"id": "D1", "col": 0, "lig": 0}  # batterie oubliée !

# Avec factory — garanti complet à chaque appel
drone = creer_drone("D1", col=0, lig=0, batterie_max=20)
```

### Principe DRY appliqué aux structures

Si la structure d'un drone change (on ajoute une clé `"niveau_carburant"`),
on modifie `creer_drone()` et c'est tout. Sans factory, il faudrait retrouver
tous les endroits où un drone est créé manuellement.

---

## Le POURQUOI — Enjeu pédagogique

Les fonctions factory introduisent deux notions fondamentales :
1. **Une fonction retourne une valeur** — ici un dict, pas juste un résultat numérique
2. **DRY appliqué aux données** — une seule source de vérité pour la structure

En phase 2, ces fonctions seront appelées par `initialiser_partie()`,
puis par les tests unitaires (C4). Elles doivent être indépendantes et
testables sans lancer le jeu.

## Les RISQUES

### Si on ne le fait pas
Tu crées des drones directement dans `initialiser_partie()` avec des dicts
littéraux. Au module P04, tu dois ajouter une clé `"bloque"`. Tu ne sais
plus combien de dicts il faut modifier.

### Si on le fait mal
Tu mutualises les instances :
```python
DRONE_VIDE = {"id": "", "col": 0, "lig": 0, "batterie": 20}
etat["drones"]["D1"] = DRONE_VIDE
etat["drones"]["D2"] = DRONE_VIDE  # DANGER : c'est le même objet !
```
Modifier `etat["drones"]["D1"]["col"]` modifie aussi D2 car ils partagent
le même dict. Toujours appeler `creer_drone()` pour obtenir un **nouvel** objet.

## Lien avec jeu/

```python
# jeu/logique.py — extraits des fonctions factory

def creer_drone(drone_id, col, lig, batterie_max):
    """Crée et retourne un nouveau dict représentant un drone."""
    return {
        "id":           drone_id,
        "col":          col,
        "lig":          lig,
        "batterie":     batterie_max,
        "batterie_max": batterie_max,
        "survivant":    None,
        "bloque":       0,
        "hors_service": False,
    }

def creer_survivant(surv_id, col, lig):
    """Crée et retourne un nouveau dict représentant un survivant."""
    return {
        "id":   surv_id,
        "col":  col,
        "lig":  lig,
        "etat": "en_attente",
    }

def creer_tempete(temp_id, col, lig, dx=1, dy=0):
    """Crée et retourne un nouveau dict représentant une tempête."""
    return {
        "id":  temp_id,
        "col": col,
        "lig": lig,
        "dx":  dx,
        "dy":  dy,
    }
```
