## Le QUOI — Définition et concept

Une **fonction factory** est une fonction dont le seul rôle est de **créer et retourner un dictionnaire initialisé**. Elle reçoit les paramètres variables (identifiant, position) et fixe elle-même les valeurs par défaut.

```python
# Sans factory : structure répétée à chaque création (fragile)
drone_1 = {"id": "D1", "col": 3, "lig": 2, "batterie": 10, "batterie_max": 20,
           "survivant": None, "bloque": 0, "hors_service": False}
drone_2 = {"id": "D2", "col": 5, "lig": 7, "batterie": 10, "batterie_max": 20,
           "survivant": None, "bloque": 0, "hors_service": False}

# Avec factory : une seule source de vérité (robuste)
def creer_drone(identifiant, col, lig):
    return {"id": identifiant, "col": col, "lig": lig,
            "batterie": BATTERIE_INIT, "batterie_max": BATTERIE_MAX,
            "survivant": None, "bloque": 0, "hors_service": False}

drone_1 = creer_drone("D1", 3, 2)
drone_2 = creer_drone("D2", 5, 7)
```

Si on décide d'ajouter un attribut `energie_solaire` aux drones, on modifie **une seule ligne** dans `creer_drone()`, et tous les drones créés partout dans le code bénéficient automatiquement de la mise à jour.

## Le POURQUOI — Enjeu pédagogique

Ce module pose la fondation de toute la logique du jeu. **Chaque fonction écrite ici sera appelée des dizaines de fois** dans `logique.py`, `console.py` et `main.py`. Une factory mal conçue (attribut manquant, type incorrect) provoque des `KeyError` et des bugs difficiles à localiser. 

Tu apprends ici le principe de **responsabilité unique** : `creer_drone()` ne fait qu'une chose — créer un drone. Elle ne place pas le drone sur la grille, ne vérifie pas les collisions, ne log pas d'événement. Ce séparation rend le code testable individuellement.

---

## Les RISQUES

### Si on ne le fait pas
La structure des dictionnaires est répétée dans le code. Une modification (ajout ou suppression d'un attribut) nécessite de retrouver et mettre à jour toutes les occurrences — garantie d'oubli et de bugs.

### Si on le fait mal

**Attribut manquant** : si `creer_drone()` oublie la clé `"bloque"`, le code plantera plus tard avec un `KeyError: 'bloque'` dans `executer_mouvement()`, et le message d'erreur ne pointera pas vers `creer_drone()`.

```python
# ❌ Attribut manquant
def creer_drone(identifiant, col, lig):
    return {"id": identifiant, "col": col, "lig": lig,
            "batterie": BATTERIE_INIT}  # oubli : bloque, hors_service, survivant...

# → KeyError: 'bloque' lors du premier appel à valider_mouvement()
```

**Référence partagée** : n'utilise jamais un dictionnaire mutable comme valeur par défaut.

```python
# ❌ Piège : tous les drones partagent le même objet None (pas un problème ici)
# mais le serait avec une liste mutable comme valeur par défaut
def creer_drone(identifiant, col, lig, historique=[]):  # ❌ NE PAS FAIRE
    return {"id": identifiant, "historique": historique}  # même liste pour tous !
```

## Lien avec `jeu/`

Les 3 fonctions factory réelles dans `jeu/logique.py` :

```python
# jeu/logique.py — lignes 47–68

def creer_drone(identifiant, col, lig):
    """Retourne un dictionnaire représentant un drone."""
    return {
        "id"          : identifiant,
        "col"         : col,
        "lig"         : lig,
        "batterie"    : BATTERIE_INIT,
        "batterie_max": BATTERIE_MAX,
        "survivant"   : None,
        "bloque"      : 0,
        "hors_service": False,
    }

def creer_tempete(identifiant, col, lig):
    """Retourne un dictionnaire représentant une tempête."""
    dx = random.choice([-1, 0, 1])
    dy = random.choice([-1, 0, 1])
    if dx == 0 and dy == 0:
        dx = 1
    return {"id": identifiant, "col": col, "lig": lig, "dx": dx, "dy": dy}

def creer_survivant(identifiant, col, lig):
    """Retourne un dictionnaire représentant un survivant."""
    return {"id": identifiant, "col": col, "lig": lig, "etat": "en_attente"}
```

Notice que `creer_tempete()` tire aléatoirement `dx` et `dy` (direction initiale de déplacement), mais garantit qu'une tempête ne reste jamais immobile (`dx=0, dy=0` est impossible).
