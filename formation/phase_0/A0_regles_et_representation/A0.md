## Le QUOI — Définition et concept

**Drone Rescue** est un jeu à deux joueurs sur une grille 12×12.

- **J1** (sauveteur) pilote des drones pour ramener des survivants à l'hôpital.
- **J2** (adversaire) déplace des tempêtes pour bloquer les drones de J1.
- Chaque tour : J1 joue, puis J2 joue, puis la météo agit automatiquement.
- **Victoire de J1** : tous les survivants sont sauvés.
- **Défaite de J1** : nombre de tours maximum dépassé, ou tous les drones hors service.

### La grille

La grille est repérée par :
- des **lettres** pour les colonnes : `A` à `L` (12 colonnes)
- des **chiffres** pour les lignes : `1` à `12` (12 lignes)

Exemple de coordonnée : `C4` = colonne C, ligne 4.

### Les symboles de la grille

| Symbole | Entité | Rôle |
|---|---|---|
| `H` | Hôpital | Base de J1, destination des survivants |
| `B` | Bâtiment | Obstacle infranchissable |
| `D` | Drone | Véhicule de sauvetage de J1 |
| `T` | Tempête | Obstacle mobile de J2 |
| `S` | Survivant | Personne à sauver (en attente) |
| `X` | Zone de danger | Case qui coûte +2 batterie |
| `.` | Case vide | Case libre |

### Les règles essentielles

- Un drone se déplace d'**au maximum 1 case** (distance de Chebyshev ≤ 1),
  diagonales incluses.
- J1 peut effectuer **au maximum 3 déplacements** par tour.
- J2 peut effectuer **au maximum 2 déplacements** de tempête par tour.
- **Coût batterie** : 1 par déplacement, +1 si transport d'un survivant,
  +2 si la case de destination est une zone X.
- **Recharge** : +3 si le drone est stationnaire ou arrive à l'hôpital.
- **Collision drone + tempête** : le drone est bloqué 2 tours.
- **Batterie à 0 en vol** : le drone est définitivement hors service.

## Les RISQUES

### Si on ne le fait pas

On code sans comprendre : on produit des fonctions qui semblent marcher
mais dont le comportement est incorrect (ex : un drone qui traverse un bâtiment,
une tempête qui ne bloque jamais).

### Si on le fait mal

On mélange les coordonnées `(colonne, ligne)` et `(ligne, colonne)`. Dans
ce jeu, la convention est `(col, lig)`. Ce bug est silencieux : le code
tourne mais les entités sont mal positionnées.

Exemple de confusion courante :
```python
# FAUX : on accède à grille[col][lig] au lieu de grille[lig][col]
grille[drone["col"]][drone["lig"]]  # ❌

# CORRECT : ligne d'abord, colonne ensuite
grille[drone["lig"]][drone["col"]]  # ✅
```

## Lien avec `jeu/`

Les symboles définis ici sont utilisés dans `jeu/config.json` :

```json
{
  "SYMBOLES": {
    "vide": ".",
    "hopital": "H",
    "batiment": "B",
    "drone": "D",
    "tempete": "T",
    "survivant": "S",
    "zone_x": "X"
  }
}
```

Et dans `jeu/affichage.py`, la grille est affichée avec des en-têtes
correspondant exactement à ce vocabulaire :

```python
# jeu/affichage.py
def afficher_grille(etat):
    """Affiche la grille 12x12 avec en-têtes colonnes (A-L) et lignes (1-12)."""
    colonnes = "  " + " ".join([chr(65 + i) for i in range(12)])
    print(colonnes)
    for lig in range(12):
        ligne_str = f"{lig + 1:2d} " + " ".join(etat["grille"][lig])
        print(ligne_str)
```
