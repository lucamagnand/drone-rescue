## Objectif
À la fin de ce module, l'étudiant comprend pourquoi les constantes ne doivent pas être dans le code, sait externaliser des valeurs dans un fichier JSON et écrire une fonction de chargement robuste.

## Le QUOI — Définition et concept

Une **valeur magique** est un nombre ou une chaîne écrit directement dans le code sans explication :

```python
# MAUVAIS — valeur magique
if batterie < 2:
    return False

# BON — constante nommée chargée depuis la config
if batterie < COUT_DEPLACEMENT_ZONE_X:
    return False
```

La **configuration externe** consiste à stocker toutes ces valeurs dans un fichier séparé (`config.json`) et à les charger au démarrage via `config.py`. Le code ne contient alors que des **noms** — jamais des littéraux.

## Le POURQUOI — Enjeu pédagogique

Si la taille de la grille change de 12×12 à 16×16, avec une configuration externalisée on modifie **un seul fichier** (`config.json`). Sans externalisation, on cherche et modifie `12` dans tout le code source — en risquant d'oublier une occurrence et de créer un bug silencieux.

## Les RISQUES

### Si on ne le fait pas
Toute modification des règles (nombre de tours, coût batterie, taille grille) nécessite de fouiller l'intégralité du code. Avec 6 fichiers et 600 lignes, une valeur `12` peut représenter la taille de la grille, un indice de liste ou une valeur de batterie — indiscernable sans contexte.

### Si on le fait mal
```python
# MAUVAIS : charger la config dans chaque fichier qui en a besoin
# logique.py
import json
with open('config.json') as f:
    cfg = json.load(f)

# affichage.py
import json
with open('config.json') as f:
    cfg = json.load(f)  # dupliqué — le fichier est ouvert deux fois
```

La bonne approche : **un seul module `config.py`** charge et expose les constantes. Tous les autres modules importent depuis `config.py`.

## Lien avec jeu/

Extrait de `jeu/config.json` :

```json
{
  "grille_taille": 12,
  "nb_drones": 3,
  "nb_tempetes": 2,
  "nb_survivants": 5,
  "batterie_max": 10,
  "tours_max": 30,
  "cout_deplacement": 1,
  "cout_zone_x": 2,
  "recharge_stationnaire": 3,
  "proba_tempete_auto": 0.5,
  "intervalle_propagation_x": 5
}
```

Extrait de `jeu/config.py` :

```python
import json
import os

def charger_config(chemin=None):
    """Charge config.json et retourne le dictionnaire de configuration."""
    if chemin is None:
        chemin = os.path.join(os.path.dirname(__file__), 'config.json')
    with open(chemin, encoding='utf-8') as f:
        return json.load(f)

_cfg = charger_config()

GRILLE_TAILLE         = _cfg['grille_taille']
BATTERIE_MAX          = _cfg['batterie_max']
TOURS_MAX             = _cfg['tours_max']
COUT_DEPLACEMENT      = _cfg['cout_deplacement']
COUT_ZONE_X           = _cfg['cout_zone_x']
RECHARGE_STATIONNAIRE = _cfg['recharge_stationnaire']
```

Usage dans `jeu/logique.py` :
```python
from config import GRILLE_TAILLE, COUT_ZONE_X
```
