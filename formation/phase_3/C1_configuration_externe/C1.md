# Module C1 — Configuration externe

## Positionnement dans le parcours

- **Requis par** : C4 (tester ses fonctions — les constantes de test viennent de la config)

- **Durée estimée** : 35 min

## Objectif
L'étudiant est capable d'identifier les valeurs magiques dans du code, de les externaliser dans un fichier JSON, et d'expliquer pourquoi cette séparation code/configuration est indispensable pour maintenir un programme sur le long terme.

## Le QUOI — Définition et concept
Une **valeur magique** (*magic number*) est un nombre ou une chaîne écrit directement dans le code sans explication :

```python
# ❌ Valeurs magiques
if batterie < 0:      # Pourquoi 0 et pas 1 ?
    drone["hors_service"] = True
if abs(dx) > 1:       # Pourquoi 1 ?
    return False
grille = [['.'] * 12 for _ in range(12)]  # Pourquoi 12 ?
```

La configuration externe déplace ces valeurs dans `config.json`, et `config.py` les lit une seule fois au démarrage pour les exposer comme **constantes nommées** :

```python
# ✅ Constantes nommées depuis config.py
from config import GRILLE_TAILLE, BATTERIE_MIN, DISTANCE_MAX
if batterie <= BATTERIE_MIN:
    drone["hors_service"] = True
grille = [['.' ] * GRILLE_TAILLE for _ in range(GRILLE_TAILLE)]
```

## Le POURQUOI — Enjeu pédagogique
Avec des valeurs magiques :
- Changer la taille de la grille de 12 à 16 = chercher et remplacer `12` partout dans le code (risque d'en rater)
- Impossible de distinguer un `1` qui signifie « distance max » d'un `1` qui signifie « indice de liste »

Avec `config.json` :
- Un seul endroit à modifier pour toutes les règles du jeu
- Le code est **auto-documenté** : `BATTERIE_MAX` se comprend sans commentaire

## Les RISQUES
### Si on ne le fait pas
Les règles du jeu sont dispersées dans 6 fichiers. Modifier une règle = 30 minutes de chasse aux valeurs magiques avec risque d'introduction de bugs.

### Si on le fait mal
**Bug typique** : charger `config.json` à chaque appel de fonction au lieu d'une seule fois au démarrage → lecture disque à chaque mouvement = ralentissement, et `config.json` peut changer entre deux lectures.

```python
# ❌ NE PAS FAIRE — lecture à chaque appel
def valider_mouvement(etat, drone_id, col, lig):
    config = json.load(open('config.json'))  # lecture disque à chaque validation !
    taille = config["grille_taille"]
    ...

# ✅ FAIRE — lecture une seule fois au démarrage (dans config.py)
import json
with open('config.json', 'r', encoding='utf-8') as f:
    _cfg = json.load(f)
GRILLE_TAILLE = _cfg["grille_taille"]  # constante disponible à l'import
```

## Lien avec jeu/
```json
// jeu/config.json — extrait des clés principales
{
  "grille_taille": 12,
  "batterie_max": 20,
  "nb_drones": 3,
  "nb_survivants": 4,
  "nb_tempetes": 2,
  "tours_max": 30,
  "max_deplacements_j1": 3,
  "max_deplacements_j2": 2,
  "cout_batterie_base": 1,
  "cout_batterie_zone_x": 2,
  "recharge_stationnaire": 3
}
```

```python
# jeu/config.py — lecture unique au démarrage
import json
import os

_CHEMIN = os.path.join(os.path.dirname(__file__), 'config.json')
with open(_CHEMIN, 'r', encoding='utf-8') as _f:
    _cfg = json.load(_f)

# Constantes exposées (en MAJUSCULES par convention PEP 8)
GRILLE_TAILLE          = _cfg["grille_taille"]
BATTERIE_MAX           = _cfg["batterie_max"]
NB_DRONES              = _cfg["nb_drones"]
TOURS_MAX              = _cfg["tours_max"]
MAX_DEPLACEMENTS_J1    = _cfg["max_deplacements_j1"]
MAX_DEPLACEMENTS_J2    = _cfg["max_deplacements_j2"]
```
