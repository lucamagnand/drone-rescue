## Le QUOI — Définition et concept

Un **test unitaire** est un petit programme qui appelle une fonction avec
des données connues et vérifie que le résultat est celui attendu.

```python
# Exemple minimal
def additionner(a, b):
    return a + b

# Test
assert additionner(2, 3) == 5
assert additionner(0, 0) == 0
assert additionner(-1, 1) == 0
print("Tous les tests passent.")
```

Si `assert` reçoit une condition **fausse**, il lève une `AssertionError` :
le programme s'arrête et tu sais exactement quel test a échoué.

> *Règle d'or* : un test = une fonction + un état contrôlé + une vérification.

---

## Le POURQUOI — Enjeu pédagogique

Sans tests, la seule façon de vérifier que le jeu fonctionne est de le lancer,
de jouer plusieurs tours et d'espérer que le bug se manifeste. C'est lent,
infiable, et impossible à automatiser.

Avec des tests, tu peux :
- vérifier `valider_mouvement()` en 0,1 seconde, sans lancer le jeu
- reproduire un bug exact (ex : batterie insuffisante avec zone X)
- être alerté automatiquement si une modification casse quelque chose

---

## Les RISQUES

### Si on ne le fait pas
Toute modification du code devient risquée : tu ne sais pas si la règle
de batterie fonctionne encore après avoir modifié `executer_mouvement()`.
Les bugs se découvrent en partie, pas avant.

### Si on le fait mal
Des tests qui testent toujours `True` (faux positifs) ne détectent rien.
Exemple piège :
```python
# Mauvais test : assert sans condition utile
assert valider_mouvement(etat, drone, (1, 0))  # retourne un tuple, toujours truthy !

# Bon test
ok, _ = valider_mouvement(etat, drone, (1, 0))
assert ok
```

### Extrait réel — `tests/test_logique.py`

```python
# Helper : état minimal et reproductible
def _etat_minimal():
    etat = initialiser_partie()
    drone = list(etat["drones"].values())[0]
    drone["col"], drone["lig"] = 0, 0
    etat["grille"][0][0] = 'D'
    return etat, drone


def test_valider_mouvement_ok():
    """Un déplacement d'une case valide est accepté."""
    etat, drone = _etat_minimal()
    drone["col"], drone["lig"] = 3, 3
    ok, raison = valider_mouvement(etat, drone, (4, 3))
    assert ok, f"Attendu True, got False : {raison}"


def test_valider_mouvement_batterie_insuffisante():
    """Un drone avec 1 batterie ne peut pas entrer en zone X (coût = 1+2 = 3)."""
    etat, drone = _etat_minimal()
    drone["col"], drone["lig"] = 3, 3
    drone["batterie"] = 1
    cible = (4, 3)
    etat["zones_x"].add(cible)
    ok, raison = valider_mouvement(etat, drone, cible)
    assert not ok
    assert "insuffisant" in raison.lower()
```

### Exécution

```bash
# Avec pytest (recommandé)
pytest tests/test_logique.py -v

# Sans pytest
python tests/test_logique.py
```

### Structure d'un bon test

```
1. ARRANGE — préparer l'état (drone, etat, valeurs)
2. ACT     — appeler la fonction testée
3. ASSERT  — vérifier le résultat
```

Ce patron s'appelle **AAA** (*Arrange, Act, Assert*). Il s'applique à
chaque test du fichier `test_logique.py`.
