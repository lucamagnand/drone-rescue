# C2 — Déboguer avec le logger

## Concept

Un **logger** est un module qui enregistre les événements du programme dans un fichier texte, indépendamment de l'affichage console. Cela permet de :
- **Tracer** l'exécution d'un programme sans interrompre le jeu
- **Reproduire un bug** : le log garde la preuve de ce qui s'est passé
- **Analyser** le comportement sur une longue partie

> *Règle d'or* : on ne débogue pas à coup de `print()`. On lit le log.

---

## Dans Drone Rescue — `jeu/logger.py`

Le module crée deux fichiers à chaque partie :

| Fichier | Rôle |
|---|---|
| `partie.log` | Journal de tous les événements (tour par tour) |
| `resultats.txt` | Bilan final : score, drones, survivants |

### Extrait réel — `jeu/logger.py` lignes 1–35

```python
import os

_DOSSIER = os.path.dirname(os.path.abspath(__file__))
_CHEMIN_LOG = os.path.join(_DOSSIER, "partie.log")
_CHEMIN_RESULTATS = os.path.join(_DOSSIER, "resultats.txt")


def demarrer_log():
    """Crée (ou recrée) le fichier de log au début de la partie."""
    with open(_CHEMIN_LOG, "w", encoding="utf-8") as f:
        f.write("=== DRONE RESCUE — Journal de partie ===\n")
        f.write("Tour | Entité | Mouvement | Événement\n")
        f.write("-" * 50 + "\n")


def enregistrer_log(ligne):
    """Ajoute une ligne au journal de la partie."""
    with open(_CHEMIN_LOG, "a", encoding="utf-8") as f:
        f.write(ligne + "\n")
```

### Points clés

- `os.path.dirname(os.path.abspath(__file__))` : le log se crée **à côté du script**, peu importe le dossier de travail courant.
- Mode `"w"` dans `demarrer_log()` : **recrée** le fichier à chaque partie (pas d'accumulation infinie).
- Mode `"a"` dans `enregistrer_log()` : **ajoute** sans écraser.
- `encoding="utf-8"` : obligatoire pour les caractères accentués (`é`, `è`, `ô`…).

---

## Schéma de flux

```
main.py
  │
  ├─ demarrer_log()          ← début de partie : crée partie.log
  │
  ├─ [boucle de jeu]
  │     └─ enregistrer_log() ← chaque événement notable
  │
  └─ sauvegarder_resultats() ← fin de partie : crée resultats.txt
```

## Exercice guidé

Tu vas enrichir le logger pour ajouter :
1. Un **horodatage** (heure de chaque événement)
2. Des **niveaux de sévérité** (`INFO`, `AVERTISSEMENT`, `ERREUR`)
3. Un **filtre** : n'écrire dans le fichier que les lignes d'un certain niveau
