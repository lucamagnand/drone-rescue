## Objectif
À la fin de ce module, l'étudiant sait **identifier systématiquement** les
cas limites d'une règle métier — avant d'écrire une seule ligne de code —
et transformer chaque cas limité en une assertion testable.

## Le QUOI — Définition et concept

Un **cas limite** (ou *edge case*) est une valeur d'entrée située à la
frontière d'une règle. Ce sont les valeurs qui font le plus souvent planter
un programme — et les valeurs que les débutants oublient de tester.

### Méthode : les 4 frontières à chercher systématiquement

| Frontière | Question à se poser | Exemple dans Drone Rescue |
|---|---|---|
| **Valeur nulle / zéro** | Que se passe-t-il si la valeur est 0 ? | batterie = 0 |
| **Valeur maximale** | Que se passe-t-il au maximum autorisé ? | case L12 (col=11, lig=11) |
| **Débordement** | Que se passe-t-il une unité au-delà du max ? | col=12 ou lig=12 — hors grille |
| **Conflit** | Deux règles s'appliquent en même temps ? | case occupée par un survivant ET une tempête |

### Exemple : règle "un drone peut se déplacer vers une case cible"

```
Cas nominaux (doivent réussir)
  ✓ Case cible libre dans la grille
  ✓ Case cible = hôpital (autorisé)
  ✓ Case cible occupée par un survivant (ramassage)

Cas limites (doivent échouer ou avoir un comportement spécial)
  ✗ Case cible hors grille (col < 0, col > 11, lig < 0, lig > 11)
  ✗ Case cible = bâtiment (bloqué)
  ✗ Case cible occupée par une autre tempête (impossible)
  ✗ Drone sans batterie (bat = 0)
  ✗ Drone hors service (déjà désactivé)
  ⚠ Case cible = zone X (autorisé mais coût supplémentaire)
```

## Le POURQUOI — Enjeu pédagogique

En P04 (validation de mouvement), tu devras écrire une fonction qui vérifie
la validité d'un déplacement. Si tu n'as pas listé les cas limites **avant**
de coder, tu en oublieras. Et c'est exactement là que les bugs s'installent.

> « Un code qui gère le cas nominal est incomplet.
> Un code qui gère tous les cas limites est robuste. »

## Les RISQUES

### Si on ne le fait pas
```python
# Fonction naïve : oublie la batterie, les bâtiments, et les bords
def valider_mouvement(etat, id_drone, col_cible, lig_cible):
    return True  # toujours valide ? ✗
```

Ce code passe sur le cas nominal mais plante dès qu'un drone essaie
de traverser un mur ou de se déplacer avec 0 de batterie.

### Si on le fait bien
```python
def valider_mouvement(etat, id_drone, col_cible, lig_cible):
    drone = etat['drones'][id_drone]
    # Cas limite 1 : hors grille
    if not (0 <= col_cible <= 11 and 0 <= lig_cible <= 11):
        return False
    # Cas limite 2 : batterie insuffisante
    if drone['batterie'] <= 0:
        return False
    # Cas limite 3 : bâtiment
    if etat['grille'][lig_cible][col_cible] == 'B':
        return False
    # ... autres cas ...
    return True
```
