## Objectif
À la fin de ce module, l'étudiant sait donner des entrées à une fonction Python,
calculer sa sortie attendue à la main — sans exécuter le code —,
et détecter un écart entre le résultat attendu et le résultat obtenu.

## Le QUOI — Définition et concept

**Simuler une fonction à la main** = jouer le rôle de Python :
en partant des entrées (paramètres), exécuter chaque ligne mentalement
et noter l'état des variables après chaque instruction.

Ce n'est pas une lecture de code. C'est une **exécution sur papier**.

### Format de trace

```
Fonction : position_depuis_chaine("B3")

Entrée : chaine = "B3"

Étape 1 : lettre = chaine[0]             → lettre = 'B'
Étape 2 : num    = int(chaine[1:])        → num = 3
Étape 3 : col    = LETTRES.index('B')     → col = 1
Étape 4 : lig    = 3 - 1                  → lig = 2
Étape 5 : return (col, lig)

Sortie attendue : (1, 2)
```

La **sortie attendue** est ce que tu calcules.
La **sortie obtenue** est ce que Python produit (`print()` ou `assert`).
Si elles diffèrent, tu as trouvé un bug.

## Le POURQUOI — Enjeu pédagogique

Sans simulation manuelle, deux problèmes arrivent systématiquement :
1. L'étudiant écrit du code sans savoir ce qu'il attend — il ne peut pas savoir si ça marche.
2. L'étudiant lance le code, voit "ça ne marche pas", et modifie au hasard.

Simuler à la main force à **prédire avant d'exécuter**.
C'est la fondation de tout débogage rigoureux.

## Les RISQUES

### Si on saute cette étape
- On ne sait jamais si un résultat correct est dû au code ou à la chance.
- On ne peut pas écrire d'assertion (`assert`) car on ne connaît pas la valeur attendue.

### Si on le fait mal
```python
# PIÈGE : confondre l'index de LETTRES avec la valeur affichée
LETTRES = "ABCDEFGHIJKL"
LETTRES.index('A')  # donne 0, pas 1
LETTRES.index('L')  # donne 11, pas 12
```

L'affichage utilise `A1` (1-based), mais le code travaille avec `(0, 0)` (0-based).
Ne pas mélanger les deux lors de la simulation.

## Lien avec jeu/

Fonctions de `jeu/logique.py` à simuler dans ce module :

```python
LETTRES = "ABCDEFGHIJKL"  # 12 lettres pour grille 12x12

def position_depuis_chaine(chaine):
    """Convertit 'B3' en (col=1, lig=2)."""
    lettre = chaine[0]
    num    = int(chaine[1:])
    col    = LETTRES.index(lettre)
    lig    = num - 1
    return (col, lig)

def chaine_depuis_position(col, lig):
    """Convertit (col=1, lig=2) en 'B3'."""
    return f"{LETTRES[col]}{lig + 1}"

def distance_chebyshev(col1, lig1, col2, lig2):
    """Distance de Chebyshev entre deux cases (déplacements en 8 directions)."""
    return max(abs(col2 - col1), abs(lig2 - lig1))
```

> Ces 3 fonctions sont les briques de base de toute la logique du jeu.
> Les maîtriser à la main rend P04 (validation) beaucoup plus fluide.
